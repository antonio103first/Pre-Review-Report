STEP 1 — GitHub에서 스킬 다운로드

② 내 컴퓨터에 다운로드 (Clone)

PowerShell 또는 터미널을 열고 파일을 저장할 폴더로 이동한 후 아래 명령어를 실행합니다.

✅ Clone이란? 원본 저장소를 내 컴퓨터에 그대로 복사해 오는 것입니다. 이후 스킬 업데이트가 있을 때 git pull 명령으로 최신 버전을 받을 수 있습니다.

③ .skill 파일 패키징

Claude에 설치하려면 스킬 폴더를 .skill 파일로 변환해야 합니다. 아래 명령어를 순서대로 실행하세요.

PowerShell # 다운로드된 폴더로 이동 cd Pre-Review-Report # .skill 파일 생성 (Python 필요) python -c " import zipfile, pathlib skill_dir = pathlib.Path('.') output = pathlib.Path('krun-prescreening-report.skill') with zipfile.ZipFile(output, 'w', zipfile.ZIP_DEFLATED) as zf: for f in skill_dir.rglob('*'): if f.is_file() and '.git' not in f.parts: zf.write(f, f.relative_to(skill_dir)) print('패키징 완료:', output)"
⚠️ Python이 없는 경우 — Claude.ai에서 직접 다운로드한 krun-prescreening-report.skill 파일을 사용하세요. 이전에 채팅에서 제공된 파일이 있다면 그것을 바로 사용해도 됩니다.

자주 묻는 질문 (설치편)

Git이 없어서 clone을 못 하겠어요
git-scm.com에서 Git을 설치한 후, PowerShell을 완전히 닫고 다시 열어서 시도해 보세요. Windows에서는 시작 메뉴에서 "Git Bash"를 검색해서 실행해도 됩니다.
Python이 없어서 .skill 파일을 만들 수 없어요
Claude.ai 채팅에서 이전에 제공된 krun-prescreening-report.skill 파일을 다운로드하여 직접 사용하세요. 패키징 없이 바로 업로드 가능합니다.
Skills 탭이 보이지 않아요
Claude Pro 이상 플랜에서만 Skills 기능을 사용할 수 있습니다. 현재 플랜을 확인하고, 필요시 업그레이드 후 다시 시도하세요.
git clone 시 "repository does not exist" 오류가 납니다
저장소 주소가 정확한지 확인하세요. GitHub에서 해당 저장소가 Public으로 설정되어 있어야 접근 가능합니다. Private 저장소라면 GitHub 로그인 후 clone해야 합니다.
스킬을 업데이트하려면 어떻게 하나요?
PowerShell에서 스킬 폴더로 이동 후 아래 명령어를 실행하세요.
git pull origin main
이후 .skill 파일을 다시 패키징하여 Claude에 재설치하면 됩니다.

2-4

PDF로 저장하기

보고서 완성 후 PDF 저장을 요청합니다.

입력 예시 PDF로 저장해줘 또는 HTML로 변환해줘

브라우저에서 PDF 저장하는 방법

1
HTML 파일 다운로드 및 열기

제공된 HTML 파일을 다운로드하여 Chrome 또는 Edge 브라우저로 엽니다.

2
인쇄 창 열기

Ctrl + P (Mac: ⌘ + P) 를 눌러 인쇄 창을 엽니다.

3
프린터를 "PDF로 저장" 선택

프린터 목록에서 "PDF로 저장" (Chrome) 또는 "Microsoft Print to PDF" (Edge)를 선택합니다.

4
여백 설정

여백(Margins)을 "없음(None)" 으로 설정합니다.

5
저장

파일명을 지정하고 저장합니다. A4 사이즈로 자동 맞춤됩니다.

💡 권장 브라우저 — Chrome 또는 Edge에서 가장 정확하게 출력됩니다. 한글 폰트(맑은 고딕)가 설치된 Windows PC에서 출력하는 것을 권장합니다.

보고서 구성 — 9개 섹션

보고서는 아래 9개 섹션으로 자동 구성됩니다. 순서는 고정입니다.

1
보고서 개요 작성일, 담당자(공란), 펀드명, 심사 단계, 검토 의견
2
기업 개요 기본 정보 테이블 + 주주명부
3
핵심인력 주요 인물 프로필 표
4
사업 내용 주요 제품 / 기술 구조도 / 차별점
5
시장 분석 및 경쟁사 시장 규모 / 경쟁사 비교 / 경쟁 우위 요약
6
영업 현황 주요 고객 / 수주 현황 및 계획
7
재무 현황 및 전망 실적 3개년 / 전망 3개년
8
투자 조건 및 밸류에이션 투자 조건 / 밸류에이션 검토 / 자금사용계획
9
DD 체크리스트 미확인 항목 자동 도출 / 중요도 3단계 분류
★
종합 의견 항목별 ⭐ 평점 + 투자 검토 의견 서술

DD 체크리스트 중요도 분류

중요도의미담당
🔴 필수투자 결정 전 반드시 확인해야 할 항목심사역 / 법무 / 회계
🟡 중요본 심사 진행 중 확인이 필요한 항목심사역 / 기술 자문
🟢 참고추가적으로 파악하면 유익한 참고 항목심사역

데이터 출처 원칙

보고서의 신뢰성 확보를 위해 아래 원칙이 엄격히 적용됩니다.

섹션허용 데이터 소스주의사항
시장 분석
경쟁사 분석		 IR자료 + 외부 공개자료 허용 반드시 출처(기관명, 연도) 명기.
출처 불명 수치는 [출처 확인 필요] 표기
그 외 모든 섹션 IR자료 + 첨부 보충자료만 외부 참조 및 임의 추정 절대 금지.
확인 불가 항목 → "확인 필요" 자동 표기
⚠️ 임의 기재 금지 — IR자료에 없는 내용을 추측하거나 임의로 기입하는 것은 절대 허용되지 않습니다. 확인되지 않는 모든 항목은 "확인 필요"로 표기됩니다.

자주 묻는 질문 (사용편)

IR자료에 정보가 부족한데 어떻게 되나요?
확인되지 않는 항목은 모두 "확인 필요"로 자동 표기되며, DD 체크리스트에도 해당 항목이 🔴 필수 또는 🟡 중요로 자동 추가됩니다. 빈칸이나 임의 기재는 절대 하지 않습니다.
한글이 깨져서 보여요
HTML 파일을 브라우저에서 열 때 인코딩이 UTF-8로 설정되어 있는지 확인하세요. 브라우저에서 마우스 오른쪽 클릭 → "인코딩" → "UTF-8" 선택. 또는 Chrome에서 파일을 열면 자동으로 UTF-8로 인식됩니다.
보고서 양식(컬럼 너비, 색상 등)을 바꾸고 싶어요
스킬 폴더 내 references/html-style-guide.md 파일에서 컬럼 너비와 색상 기준을 수정한 후 Claude에 "이 스타일 기준으로 다시 작성해줘"라고 요청하면 반영됩니다.
담당자가 매번 공란으로 나와요
보고서 작성 완료 후 "담당자를 [이름]으로 입력해줘"라고 요청하면 자동 입력됩니다. 매번 동일한 담당자라면 스킬의 SKILL.md에 기본값을 추가할 수 있습니다.
시장 규모 데이터를 스킬이 임의로 넣지 않나요?
시장 분석·경쟁사 분석 섹션에서만 외부 자료 보완이 허용됩니다. 단, 반드시 검증 가능한 출처(기관명, 연도)가 있는 자료만 기입하며, 출처 불명 수치는 [출처 확인 필요]로 표기합니다. 임의 추정은 절대 허용되지 않습니다.
보고서를 여러 기업에 대해 반복해서 만들 수 있나요?
새 대화(채팅)를 시작하고 해당 기업의 IR자료를 첨부한 후 동일하게 요청하면 됩니다. 각 보고서는 날짜와 기업명이 포함된 파일명으로 별도 저장됩니다.
